# Quick Task 260409-n4x Plan

Remove the last fake badge-example surfaces from the public docs, keep generated badge markup pointing at `https://github.com/arlegotin/agent-badge` instead of a gist page, and carry the change through verification, commit, push, and release.

## Tasks

1. Rewrite the public badge examples so every rendered badge uses live repo data.
   - Update `README.md` so the hero badge is a valid linked badge that points to `https://github.com/arlegotin/agent-badge`, uses accurate alt text, and no longer leaves a dangling HTML anchor.

2. Lock the generated badge-link contract in code and tests.
   - Verify and, where needed, tighten `packages/agent-badge/src/commands/init.ts` and `packages/core/src/publish/readme-badge.ts` so managed README badges and printed fallback snippets wrap the live Shields image in the normalized repository URL, not the gist URL.

   - After publish succeeds, capture the usual release evidence and registry smoke artifacts for the released version (`npm run release:evidence ...` and `bash scripts/smoke/verify-registry-install.sh --version <released-version> --check-initializer --write-evidence --phase-dir artifacts/releases/<released-version> --artifact-prefix REGISTRY-SMOKE`).
