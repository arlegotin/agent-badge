# Quick Task 260409-jn2 Plan

## Goal

Make shared-runtime validation robust across current and older installed CLIs by adding a backward-compatible probe path, while preserving explicit diagnostics and shipping the change through tests, docs, and release/push flow.

## Tasks

1. Implement a backward-compatible shared-runtime probe without weakening diagnostics.
   - Update `packages/core/src/runtime/shared-cli.ts` so runtime inspection probes `agent-badge --version` first, then falls back to a legacy-compatible probe (`agent-badge version`) when the first probe fails in a compatibility-pattern way.
   - Keep status semantics stable: `missing` only for command-resolution failures, `available` only for successful probe output, and `broken` for all other failures.
   - Preserve and improve diagnostic detail for broken states by carrying forward explicit stderr/stdout/error context (including both probe attempts when fallback is used).
   - Validate that `packages/agent-badge/src/commands/init.ts` and `packages/agent-badge/src/commands/config.ts` continue to surface the same remediation language and setup-status messaging contracts.

2. Add regression coverage and synchronize docs/checks to the new probe contract.
   - Extend `packages/core/src/runtime/shared-cli.test.ts` with success/fallback/failure cases covering: direct `--version` success, fallback `version` success, missing runtime, and dual-probe broken diagnostics.
   - Adjust command tests only where needed (for example `packages/agent-badge/src/commands/init.test.ts` and `packages/agent-badge/src/commands/config.test.ts`) to lock the expected operator-facing lines for missing vs broken runtime states.
   - Update `docs/INSTALL.md`, `docs/QUICKSTART.md`, and `docs/TROUBLESHOOTING.md` so recovery guidance reflects the backward-compatible probe behavior and still recommends upgrading to latest shared runtime.
   - Update `scripts/verify-docs.sh` assertions for any wording or invariant changes introduced above.
   - Verify with:
     - `npm test -- --run packages/core/src/runtime/shared-cli.test.ts packages/agent-badge/src/commands/init.test.ts packages/agent-badge/src/commands/config.test.ts`
     - `npm run docs:check`

3. Prepare release metadata and push path for publication.
   - Add release notes entry for this fix (`CHANGELOG.md` and a changeset if required by the current release process) describing the backward-compatible runtime probe hardening and diagnostic preservation.
   - Run release gates relevant to this scope: `npm run build`, `npm run typecheck`, and `npm run release:preflight`.
   - Commit with a release-ready message, push the branch, and document that merging to `main` triggers `.github/workflows/release.yml`.
   - After publish, follow `docs/maintainers/RELEASE.md` evidence/smoke steps (`release:evidence` and `scripts/smoke/verify-registry-install.sh --check-initializer --write-evidence`) for the released version.
