## Threat Model

### Assets
- truthful release-readiness reporting
- operator decision quality before publish
- consistency between tooling output, docs, and planning state

### Threats
- **High:** preflight still collapses materially different external blockers into vague summaries, causing false readiness claims
- **High:** docs imply that local rehearsal equals production readiness
- **Medium:** planning-state wording drifts from tooling and docs, reviving tribal knowledge

### Required mitigations
- Emit or document explicit blocker categories for npm auth, registry version drift, same-version conflict, package ownership confirmation, and trusted-publisher confirmation.
- Preserve the documented limitation that trusted-publisher state cannot be auto-proved locally if that remains true after implementation.
- Keep `.planning/STATE.md` blocker wording aligned with the new release vocabulary.
</threat_model>
