<threat_model>
## Threat Model

### Assets
- truthful production publish proof
- release version integrity across source, workflow, and npm
- local uncommitted work that must not be disrupted during reconciliation

### Threats
- **High:** a duplicate publish caused by dispatching from the stale `1.1.2` checkout before reconciling the already-visible `1.1.3` release
- **High:** false evidence that cites the wrong GitHub Actions run, commit, or version as the canonical trusted publish
- **Medium:** accidental disturbance of unrelated local work while inspecting or checking out the release commit

### Required mitigations
- Reconcile the release candidate first; no fresh publish attempt until one exact run/commit/version tuple is established or explicitly proven missing.
- Record the release decision, run metadata, and publish path in one phase-owned artifact with exact values, not approximations.
- Use a clean temporary worktree or equivalent safe checkout if the released commit must be inspected locally.
</threat_model>
