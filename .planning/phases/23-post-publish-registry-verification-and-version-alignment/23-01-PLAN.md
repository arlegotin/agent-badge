## Threat Model

### Assets
- truthful post-publish registry verification for `1.1.3`
- phase-owned evidence artifacts
- privacy-safe smoke evidence that does not leak local scratch paths

### Threats
- **High:** Phase 23 smokes the stale local `1.1.2` state instead of the actually shipped `1.1.3` release
- **High:** the smoke script writes Phase 13 filenames again, making historical and current proof ambiguous
- **Medium:** `npm init agent-badge@latest` is claimed as working without a direct clean-room run
- **Medium:** saved evidence leaks `/tmp` paths or scratch-directory details

### Required mitigations
- Derive the exact smoke version from Phase 22 publish evidence before running any installs.
- Add a backward-compatible artifact-prefix input so `23-REGISTRY-SMOKE.*` is the saved proof.
- Run and record a separate `npm init agent-badge@latest` check.
- Keep local paths in stdout/stderr only; do not persist them in Phase 23 artifacts.
</threat_model>
